# 1. エンティティモデル

## 1-1. エンティティ定義

| ノード名 | 説明 |
|---------|------|
| Movie | 映画作品 |
| Person | 人物（監督、俳優、スタッフ） |
| Organization | 制作会社・配給会社 |
| Genre | ジャンル |

## 1-2. エンティティの属性一覧

| ノード名 | 属性 | 型 | 必須 | 説明 |
|---------|------|----|----|------|
| Movie | unique_key | string | ✓ | ユニークキー（official_urlまたはtitle+release_yearから生成） |
| Movie | title | string | ✓ | 邦題 |
| Movie | title_en | string |  | 英題 |
| Movie | release_year | integer |  | 公開年 |
| Movie | duration | integer |  | 上映時間（分） |
| Movie | synopsis | string |  | あらすじ |
| Movie | jfdb_url | string |  | 日本映画データベースURL |
| Movie | official_url | string |  | 公式サイトURL |
| Person | unique_key | string | ✓ | ユニークキー（name） |
| Person | name | string | ✓ | 氏名 |
| Organization | unique_key | string | ✓ | ユニークキー（name） |
| Organization | name | string | ✓ | 組織名 |
| Organization | type | string |  | 組織種別（production/distributor） |
| Genre | unique_key | string | ✓ | ユニークキー（name） |
| Genre | name | string | ✓ | ジャンル名 |

※注意:冪等性確保のための一意キーは、名称などの属性から選定する。

# 2. ドメインモデル

## 2-1. ドメイン定義（Cypher風記法）

```cypher
// 制作体制
(Movie)-[:DIRECTED_BY]->(Person)
(Movie)-[:PRODUCED_BY]->(Person)
(Person)-[:ACTED_IN {role: "役名"}]->(Movie)
(Person)-[:STAFF_OF {position: "職種"}]->(Movie)

// 制作・配給
(Movie)-[:PRODUCTION]->(Organization)
(Movie)-[:DISTRIBUTED_BY]->(Organization)

// ジャンル分類
(Movie)-[:BELONGS_TO]->(Genre)

// 受賞関係（将来拡張）
(Movie)-[:AWARDED {award_name: "賞名", year: 年}]->(Movie)
```

## 2-2. リレーションシップ一覧

| リレーションシップ名 | StartNode | EndNode | 説明 |
|-------------------|-----------|---------|------|
| DIRECTED_BY | Movie | Person | 監督 |
| PRODUCED_BY | Movie | Person | プロデューサー |
| ACTED_IN | Person | Movie | 出演 |
| STAFF_OF | Person | Movie | スタッフ |
| PRODUCTION | Movie | Organization | 製作会社 |
| DISTRIBUTED_BY | Movie | Organization | 配給会社 |
| BELONGS_TO | Movie | Genre | ジャンル分類 |

## 2-3. リレーションシップの属性一覧

| リレーションシップ名 | StartNode | EndNode | 属性 | 型 | 説明 |
|-------------------|-----------|---------|------|----|----|
| ACTED_IN | Person | Movie | role | string | 役名 |
| STAFF_OF | Person | Movie | position | string | 職種（脚本・音楽・撮影など） |

# 3. インデックス＆制約

## インデックス

```cypher
CREATE INDEX movie_title_idx FOR (m:Movie) ON (m.title);
CREATE INDEX movie_release_year_idx FOR (m:Movie) ON (m.release_year);
CREATE INDEX person_name_idx FOR (p:Person) ON (p.name);
CREATE INDEX organization_name_idx FOR (o:Organization) ON (o.name);
CREATE INDEX genre_name_idx FOR (g:Genre) ON (g.name);
```

## 制約

```cypher
CREATE CONSTRAINT movie_unique_key_unique FOR (m:Movie) REQUIRE m.unique_key IS UNIQUE;
CREATE CONSTRAINT person_unique_key_unique FOR (p:Person) REQUIRE p.unique_key IS UNIQUE;
CREATE CONSTRAINT organization_unique_key_unique FOR (o:Organization) REQUIRE o.unique_key IS UNIQUE;
CREATE CONSTRAINT genre_unique_key_unique FOR (g:Genre) REQUIRE g.unique_key IS UNIQUE;
```

# 4. 設計メモ

## 特記事項

| 対象 | 規則 | 例 |
|------|------|-----|
| ノードラベル | 大文字始まり→CamelCase | `Movie`, `Person`, `Organization` |
| リレーションシップ | 大文字→SNAKE_CASE | `DIRECTED_BY`, `ACTED_IN` |
| プロパティ名 | 小文字→snake_case | `title`, `release_year`, `unique_key` |
| 空間データ | point型を使用 | `point({latitude: 35.6, longitude: 139.7})` |
| 時間データ | date/datetime型を使用 | `date("2024-01-01")` |

**冪等性の確保**:
- 各ノードには `unique_key` を必須属性として定義し、MERGEでの登録時に使用する
- Movieの場合: `official_url` が存在すればそれを、なければ `title + release_year` を組み合わせてunique_keyとする
- Personの場合: `name` をunique_keyとする（同姓同名の考慮は将来課題）
- Organizationの場合: `name` をunique_keyとする

**人物ノードの統合**:
- 監督、俳優、プロデューサー、スタッフは同一の `Person` ノードとして扱う
- 役割はリレーションシップの種類（DIRECTED_BY、ACTED_IN、STAFF_OF等）で表現する

**データソースの特徴**:
- CAST欄には「氏名:役名」の形式でデータが格納されている
- STAFF欄には「氏名:職種」の形式でデータが格納されている
- PRODUCER/DIRECTORは専用フィールドが存在する

## データの特徴・制約

- ソースデータはテキスト形式（KEY: VALUE）で構造化されている
- 公開年が未入力の作品が存在する（RELEASE_YEARが空）
- 上映時間（DURATION）が未入力の作品が存在する
- 複数の人物が1つの役割を担う場合がある（複数監督、複数プロデューサー）
- 制作会社名が長文の場合がある（例: 「仕掛人・藤枝梅安」時代劇パートナーズ42社）

## 今後の拡張可能性

1. **受賞情報の追加**:
   - Awardノードを追加し、(Movie)-[:AWARDED]->(Award)の関係を定義
   - 各種映画祭・賞のデータを統合

2. **時系列データの拡張**:
   - release_yearをdate型に拡張し、公開日の精度を向上
   - 制作期間、撮影期間などの時系列情報を追加

3. **ロケーション情報の追加**:
   - Locationノードを追加し、撮影地・公開館などの地理情報を統合
   - point型を使った空間検索の実装

4. **評価・レビュー情報の追加**:
   - Reviewノードを追加し、批評家や観客のレビューデータを統合
   - 評価スコア、興行収入などの指標を追加

5. **作品間の関係性**:
   - シリーズ作品、リメイク、原作などの関係を(Movie)-[:RELATED_TO]->(Movie)で表現
